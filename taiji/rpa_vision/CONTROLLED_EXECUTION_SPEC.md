# RPA Vision - 受控执行规范 v1.0

**制定时间：** 2026-03-11  
**适用范围：** 太极OS / RPA Vision 模块  
**状态：** 生效

---

## 核心原则

太极OS 的桌面执行能力遵循一个核心原则：

> **先判断，再执行；执行留痕，可回放，可追责。**

这不是建议，是硬约束。

---

## 1. 点击执行规范

### 1.1 强制规则

**Rule 1：默认走 execute_safe_click()**
- 所有真点击必须通过 `execute_safe_click()` 执行
- 禁止直接调用 `pyautogui.click()` 或其他裸点击函数
- 违反此规则的代码不允许进入生产

**Rule 2：三道闸门强制检查**
- 窗口绑定检查（必须）
- 高风险区域检查（必须）
- 目标安全性检查（必须）
- 任一闸门失败，立即拒绝执行

**Rule 3：审计日志强制记录**
- 每次点击（无论成功/失败）必须写入 `click_audit_log.jsonl`
- 日志必须包含：时间戳、目标文本、目标类型、坐标、闸门结果、执行结果
- 日志文件不允许删除，只允许归档

**Rule 4：截图强制保存**
- 每次真点击必须保存点击前后截图
- 截图命名格式：`before_click_{timestamp}.png` / `after_click_{timestamp}.png`
- 截图保存路径：`debug_screenshots/`

### 1.2 高风险区域定义

以下区域默认禁止点击：

| 区域 | 定义（相对窗口百分比） | 风险类型 |
|------|----------------------|---------|
| 顶部栏 | y ≤ 5% | 标签栏、菜单栏 |
| 底部栏 | y ≥ 95% | 任务栏、状态栏 |
| 右上角 | x ≥ 95% && y ≤ 5% | 关闭按钮 |

### 1.3 目标安全性白名单

**允许的 OCR 类型：**
- `static_text` - 静态文本
- `content_text` - 内容文本

**禁止的动作词（黑名单）：**
- 中文：提交、确认、删除、关闭、退出、取消、保存、发送
- 英文：submit、confirm、delete、close、exit、cancel、save、send

**扩展原则：**
- 新增白名单类型需要经过验证
- 新增黑名单词需要记录原因

### 1.4 执行流程

```
1. 窗口绑定检查
   ↓ pass
2. 高风险区域检查
   ↓ pass
3. 目标安全性检查
   ↓ pass
4. 截图前状态
   ↓
5. 执行点击
   ↓
6. 截图后状态
   ↓
7. 写入审计日志
   ↓
8. 返回结果
```

任一步骤失败，立即终止并记录。

## 2. 输入执行规范（safe_type）

### 2.1 强制规则

**Rule 1：默认走 execute_safe_type()**
- 所有真输入必须通过 `execute_safe_type()` 执行
- 禁止直接调用 `pyautogui.write()` 或其他裸输入函数

**Rule 2：输入内容安全检查**
- 禁止输入包含敏感信息（密码、密钥、Token）
- 禁止输入包含恶意命令（rm -rf、del /f /s /q）
- 禁止输入包含 SQL 注入、XSS 等攻击代码

**Rule 3：输入目标验证**
- 必须验证当前焦点窗口
- 必须验证输入框类型（文本框、搜索框、编辑器）
- 禁止在密码框、命令行、终端中输入

**Rule 4：审计日志强制记录**
- 每次输入必须写入 `type_audit_log.jsonl`
- 日志必须包含：时间戳、目标窗口、输入长度（不记录内容）、执行结果

### 2.2 敏感信息检测

**黑名单关键词：**
- password、passwd、pwd、secret、token、key、api_key
- 信用卡号模式（16 位数字）
- 身份证号模式（18 位数字）

**检测原则：**
- 输入内容包含黑名单关键词 → 拒绝
- 输入内容匹配敏感模式 → 拒绝
- 不确定时 → 拒绝

## 3. 快捷键执行规范（safe_hotkey）

### 3.1 强制规则

**Rule 1：默认走 execute_safe_hotkey()**
- 所有快捷键必须通过 `execute_safe_hotkey()` 执行
- 禁止直接调用 `pyautogui.hotkey()` 或其他裸快捷键函数

**Rule 2：快捷键白名单**
- 只允许执行白名单内的快捷键
- 白名单外的快捷键默认拒绝

**Rule 3：高风险快捷键禁止**
- Alt+F4（关闭窗口）
- Ctrl+W（关闭标签）
- Ctrl+Q（退出应用）
- Win+L（锁屏）
- Ctrl+Alt+Del（系统中断）

**Rule 4：审计日志强制记录**
- 每次快捷键执行必须写入 `hotkey_audit_log.jsonl`
- 日志必须包含：时间戳、快捷键组合、目标窗口、执行结果

### 3.2 快捷键白名单

**允许的快捷键：**
- Ctrl+C（复制）
- Ctrl+V（粘贴）
- Ctrl+X（剪切）
- Ctrl+Z（撤销）
- Ctrl+Y（重做）
- Ctrl+A（全选）
- Ctrl+F（查找）
- Ctrl+S（保存）- 仅在编辑器/文档中允许

**扩展原则：**
- 新增白名单快捷键需要经过风险评估
- 高风险快捷键永久禁止

## 4. Dry-Run 模式

### 4.1 强制规则

**Rule 1：新功能必须先 Dry-Run**
- 所有新增的执行功能必须先实现 Dry-Run 模式
- Dry-Run 模式必须完整模拟执行流程（除了真实执行）
- Dry-Run 模式必须输出详细日志

**Rule 2：生产环境可选 Dry-Run**
- 生产环境支持通过参数启用 Dry-Run 模式
- Dry-Run 模式下所有执行动作仅模拟，不真实执行
- Dry-Run 模式下仍然记录审计日志（标记为 dry_run）

### 4.2 Dry-Run 输出格式

```
[DRY-RUN] 所有闸门通过，模拟点击 (x, y)
[DRY-RUN] 目标文本: xxx
[DRY-RUN] 目标类型: xxx
[DRY-RUN] 闸门结果: {...}
```

## 5. 审计日志规范

### 5.1 日志文件

| 动作类型 | 日志文件 | 格式 |
|---------|---------|------|
| 点击 | click_audit_log.jsonl | JSONL |
| 输入 | type_audit_log.jsonl | JSONL |
| 快捷键 | hotkey_audit_log.jsonl | JSONL |

### 5.2 日志字段（点击）

```json
{
  "timestamp": "ISO 8601 时间戳",
  "target_text": "目标文本",
  "target_type": "目标类型",
  "target_bbox": [x1, y1, x2, y2],
  "click_point": {"x": x, "y": y},
  "gates": {
    "window_binding": "pass/fail:reason",
    "risk_zone": "pass/fail:reason",
    "target_safety": "pass/fail:reason"
  },
  "result": "success/rejected/error",
  "action": "executed/simulated/dry_run",
  "window_title_before": "窗口标题",
  "window_title_after": "窗口标题",
  "window_title_changed": true/false,
  "error": "错误信息（如果有）"
}
```

### 5.3 日志保留策略

- 日志文件不允许删除
- 日志文件每月归档一次
- 归档路径：`audit_logs/archive/YYYY-MM/`
- 归档后原文件清空，但保留文件

## 6. 截图规范

### 6.1 截图保存

**保存路径：** `debug_screenshots/`

**命名格式：**
- 点击前：`before_click_{timestamp}.png`
- 点击后：`after_click_{timestamp}.png`

**保留策略：**
- 最近 100 次执行的截图保留
- 超过 100 次的截图自动清理（保留最新）
- 异常执行的截图永久保留

### 6.2 截图内容

- 全屏截图（不裁剪）
- PNG 格式（无损）
- 包含时间戳水印（可选）

## 7. 错误处理

### 7.1 闸门失败

**处理原则：**
- 立即终止执行
- 记录失败原因
- 写入审计日志
- 返回明确错误信息

**不允许：**
- 绕过闸门
- 降级执行
- 静默失败

### 7.2 执行失败

**处理原则：**
- 记录完整错误堆栈
- 保存失败时的截图
- 写入审计日志
- 返回明确错误信息

**不允许：**
- 自动重试（除非明确配置）
- 静默失败
- 隐藏错误

---

## 8. 扩展原则

### 8.1 新增执行动作

新增任何执行动作（拖拽、滚动、右键等）必须遵循：

1. 实现三道闸门（或等价安全检查）
2. 实现审计日志
3. 实现 Dry-Run 模式
4. 实现截图保存（如果适用）
5. 通过安全评审

### 8.2 修改现有规范

修改现有规范必须：

1. 记录修改原因
2. 评估影响范围
3. 更新文档版本
4. 通知所有使用者

---

## 9. 违规处理

### 9.1 违规定义

以下行为视为违规：

1. 绕过 `execute_safe_*()` 直接调用裸函数
2. 删除或篡改审计日志
3. 禁用安全闸门
4. 在生产环境中使用未经验证的执行动作

### 9.2 违规后果

- 代码不允许进入生产
- 必须回滚到合规版本
- 记录违规事件
- 评估影响范围

## 附录：术语表

| 术语 | 定义 |
|------|------|
| 闸门 | 执行前的安全检查点，必须通过才能继续 |
| 审计日志 | 记录所有执行动作的 JSONL 格式日志文件 |
| Dry-Run | 模拟执行模式，不真实执行但完整记录 |
| 裸函数 | 直接调用底层库（pyautogui 等）的函数，未经安全封装 |
| 白名单 | 明确允许的目标类型/快捷键列表 |
| 黑名单 | 明确禁止的动作词/快捷键列表 |
| bbox | 边界框，四元组 (x1, y1, x2, y2) |
| click_point | 点击点，二元组 (x, y) |
